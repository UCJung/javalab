# Design Pattern : State

## 개요
---	
특정 객체가 해당 객체의 상태에 따라 다른 행위를 하여야 할 경우
객체의 상태를 분리하여 유연하고 확장성 있는 설계를 하고자 할 때 사용되는 패턴임.

## 전략 패턴과의 차이점
---
전략패턴의 경우 Client에서 어떠한 전략을 사용할지 결정하는 반면
State 패턴의 경우 객체의 상태에 수행될 전략이 결정됨.

## Use Case
---
1. 사용자는 주문을 생성하고 주문의 상태를 변경할 수 있다.
2. 주문은 아래와 같은 정보를 갖는다.
    * 상태 : 0, 1, 2
    * 주문상품명
    * 주문총금액
3. 주문처리에는 아래와 같은 같은 기능이 제공되어야 한다.
    * 주문의 상태를 변경한다.
    * 주문을 취소할 수 있다.
4. 고려사항
    * 각 행위는 주문의 상태에 따라 다른 로직이나 절차가 수행될 수 있다.
    
## 과제
---
1. 현재 구현된 로직을 State Pattern을 적용하여 수정하세요.
2. 추가과제

    * Strategy Pattern을 적용하여 구현
